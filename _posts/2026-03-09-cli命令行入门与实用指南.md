CLI（Command Line Interface，命令行界面）是通过输入文本命令与计算机交互的方式。虽然它看起来没有图形界面直观，但在效率、自动化和可重复操作上非常强大。

这篇文章会带你快速认识 CLI：它是什么、为什么值得学、常见命令有哪些，以及如何从新手过渡到日常高效使用。

## 1. 什么是 CLI

CLI 是一种“通过键盘输入命令来控制系统和程序”的交互方式。与 GUI（图形界面）相比，CLI 的特点是：

1. **操作快**：熟练后很多操作比鼠标点击更快。
2. **可批量处理**：一个命令可同时处理大量文件。
3. **易自动化**：命令可写成脚本，重复执行。
4. **可追溯**：命令历史便于复盘与共享。

常见 CLI 环境：

1. macOS/Linux 的 `Terminal`
2. Windows 的 `PowerShell`、`cmd`、`Windows Terminal`
3. 开发工具中的内置终端（如 VS Code Terminal）

## 2. 为什么学习 CLI

如果你经常做这些事情，CLI 会明显提升效率：

1. 管理项目文件、重命名、查找、压缩
2. 使用 Git 进行版本控制
3. 安装依赖、运行脚本、构建项目
4. 登录服务器、查看日志、排查问题
5. 自动化重复工作（备份、批量处理、定时任务）

一句话理解：GUI 适合“偶尔做”，CLI 适合“经常做”。

## 3. 常见基础命令（通用思路）

下面以类 Unix 终端（macOS/Linux）为例，Windows PowerShell 也有对应命令。

| 目标 | 常见命令 | 说明 |
|---|---|---|
| 查看当前位置 | `pwd` | 显示当前目录 |
| 列出文件 | `ls` | 可加 `-la` 查看详细信息 |
| 切换目录 | `cd <路径>` | `cd ..` 返回上一级 |
| 新建目录 | `mkdir <目录名>` | 创建文件夹 |
| 新建文件 | `touch <文件名>` | 创建空文件 |
| 复制文件 | `cp <源> <目标>` | 复制文件或目录 |
| 移动/重命名 | `mv <源> <目标>` | 也可用于改名 |
| 删除文件 | `rm <文件名>` | 删除前要确认 |
| 查看文件内容 | `cat <文件>` | 快速查看文本 |
| 搜索文本 | `grep "关键字" 文件` | 在内容中检索 |
| 查找文件 | `find . -name "*.md"` | 按条件搜索文件 |

建议：先熟练 `cd`、`ls`、`mkdir`、`cp`、`mv`、`rm`，这六个命令覆盖了大部分日常文件操作。

## 4. 三个高频 CLI 使用场景

### 4.1 项目开发

1. `git clone` 拉取项目
2. `npm install` 或 `pip install -r requirements.txt` 安装依赖
3. `npm run dev` 或 `python main.py` 启动项目

优势是整个流程可复用、可记录、可发给同事直接执行。

### 4.2 运维与排障

1. 用 `ssh` 登录远程服务器
2. 用 `tail -f` 追踪日志
3. 用 `ps`、`top`、`lsof` 查看进程和端口占用

CLI 在“快速定位问题”上通常比图形化面板更直接。

### 4.3 批量处理

1. 批量重命名文件
2. 批量转码与压缩
3. 组合命令完成流水线处理（例如 `cat` + `grep` + `sort`）

这类工作如果用 GUI 手动点，耗时且容易出错。

## 5. 初学者常见误区

1. **害怕输错命令**：先在测试目录练习，逐步建立信心。
2. **不看帮助文档**：先学会 `--help`，再学会 `man`。
3. **上来就背很多命令**：先掌握 10 个高频命令，再扩展。
4. **忽略路径概念**：相对路径与绝对路径是核心基础。
5. **直接执行来源不明命令**：尤其是带 `sudo` 或删除类命令，先理解再执行。

## 6. 现在主流的 AI CLI 工具

随着大模型能力增强，AI CLI 已经成为很多开发者的日常工具。它们通常支持代码解释、命令建议、文件修改、自动化任务拆解等能力。

下面是目前讨论度较高、使用较广的几类工具（按生态分类）：

1. **OpenAI Codex CLI**
	面向终端场景的 AI 编码助手，适合在本地项目中进行代码生成、修改和任务执行。
	链接：<https://github.com/openai/codex>

2. **Claude Code（Anthropic）**
	强调代码库理解与多步任务处理，常用于复杂重构、排查和大规模代码解释。
	链接：<https://docs.anthropic.com/>

3. **Gemini CLI（Google）**
	Google 生态下的命令行 AI 工具，适合与其模型与云服务工作流结合。
	链接：<https://ai.google.dev/>

4. **Aider**
	开源、上手快，支持在终端里让 AI 直接参与 Git 工作流（改代码、提交建议等）。
	链接：<https://github.com/Aider-AI/aider>

5. **Warp（AI Terminal）**
	不是传统“纯命令行工具”，更像带 AI 能力的现代终端，适合希望兼顾 GUI 体验和 CLI 效率的用户。
	链接：<https://www.warp.dev/>


## 7. 一句话总结

CLI 不是“更难的 GUI”，而是“更高效的工作方式”。当你的任务开始重复、规模变大、需要协作和自动化时，命令行会成为非常稳定的生产力工具。
