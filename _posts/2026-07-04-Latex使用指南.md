写论文、写简历、排一本几百页的书，用 Word 排到最后往往会遇到公式编号乱掉、图表引用对不上、格式莫名其妙跑偏这些糟心事。LaTeX 就是为了解决这类问题而生的——它不是"所见即所得"的文字处理器，而是"所想即所得"：你专心写内容，排版交给引擎去处理，公式、编号、引用这些细节它比 Word 稳得多。这篇文章介绍 LaTeX 是什么、怎么上手、常用语法有哪些。

## 什么是 LaTeX

LaTeX 是基于 TeX 排版引擎的一套文档准备系统。写 LaTeX 的过程和写代码很像：你在 `.tex` 源文件里用标记语法描述文档结构（章节、公式、表格、引用……），再用编译器把它编译成排版精美的 PDF。

它和 Word 最大的区别在于**内容与排版是分离的**：Word 里样式和内容混在一起，改起来牵一发动全身；LaTeX 里你只管写内容、打标记，标题层级、编号、引用格式统一由文档模板和引擎控制，整篇文档的排版风格天然保持一致。这也是为什么数学、物理、计算机等领域的论文和期刊几乎都用 LaTeX——公式多、引用多、格式要求严格，正好是它最擅长的场景。

## 三种上手方式

### 方式一：Overleaf（云端，最快上手）

- 官网：[overleaf.com](https://www.overleaf.com/)

Overleaf 是一个免费的在线 LaTeX 编辑器，浏览器打开就能写，不需要在本地安装任何东西，自带实时编译预览，还支持多人协作（体验很像 Google Docs 那种共享文档），并且提供海量的期刊 / 会议 / 简历模板，直接选模板开始写就行。免费版对协作人数、编译时间有一些限制，但个人写论文、作业完全够用。

**如果是第一次接触 LaTeX，强烈建议从 Overleaf 开始**，不用折腾环境配置，专心体验 LaTeX 本身的语法就够了。

### 方式二：完整发行版 + 独立编辑器（传统全家桶）

这是最经典的本地方案：先安装一套完整的 TeX 发行版——Windows / Linux 上通常装 **[TeX Live](https://tug.org/texlive/)**，macOS 上装 **[MacTeX](https://tug.org/mactex/)**（本质也是 TeX Live 的 macOS 打包版）；装好之后，再配一个专门写 LaTeX 的编辑器，比如 **[TeXstudio](https://www.texstudio.org/)**、**[TeXmaker](https://www.xm1math.net/texmaker/)**，这些编辑器内置了编译按钮、语法高亮、公式实时预览等功能，开箱即用。

优点是功能完整、不依赖网络、编译速度稳定；缺点是发行版体积不小（完整版 TeX Live 能到好几个 G），第一次安装会比较耗时。

### 方式三：VS Code + LaTeX Workshop 插件（现代轻量）

这条路同样需要先装一套 TeX 发行版（可以选精简版，比如 **[TinyTeX](https://yihui.org/tinytex/)**，用到什么宏包再按需安装，省去一次性下载完整版的体积），然后在 VS Code 里装上 **[LaTeX Workshop](https://marketplace.visualstudio.com/items?itemName=James-Yu.latex-workshop)** 插件。

好处是可以把 LaTeX 项目和其他代码项目放在同一个编辑器里统一管理，插件功能也很齐全：实时预览、跳转定义、自动补全、SyncTeX 双向同步（点 PDF 跳源码、点源码跳 PDF 对应位置）。如果你本来就习惯用 VS Code 写代码，这条路会顺手很多——第一次配置稍微麻烦一点，配好之后体验非常流畅。

## LaTeX 的好处和典型用途

- **精美的数学公式排版**：这是 LaTeX 最著名的强项。行内公式用 `$...$`，独立公式用 `equation` 环境，编号、对齐、多行公式都有成熟的语法支持，一致性和美观度是 Word 公式编辑器很难比的。
- **交叉引用和参考文献自动化**：`\label`、`\ref`、`\cite` 配合 BibTeX / BibLaTeX，引用文献、图表、公式编号全部自动生成、自动更新，不会出现 Word 里手动改编号却漏改几处的情况。
- **官方模板即插即用**：几乎所有学术期刊、会议都提供官方 LaTeX 模板（`.cls`、`.sty` 文件），下载模板、往里面填内容，格式就自动符合投稿要求。
- **大型文档的一致性**：书籍、毕业论文这种几百页的文档，LaTeX 能保证标题层级、页码、目录风格高度统一，不容易因为手抖或者复制粘贴搞乱格式。
- **纯文本源码，对版本控制友好**：`.tex` 文件就是纯文本，非常适合用 Git 做版本管理，每次改动的 diff 一目了然，这是 Word 的二进制格式完全做不到的。
- **典型用途**：学术论文、毕业论文、书籍排版、简历（CV）、Beamer 幻灯片（可以理解成 LaTeX 版的 PowerPoint）。

## 常见命令速查

### 基本文档结构

```latex
\documentclass{article}
\usepackage[utf8]{inputenc}
\usepackage{amsmath, amssymb} % 数学公式增强
\usepackage{graphicx}          % 插图
\usepackage{hyperref}          % 超链接与交叉引用

\title{文档标题}
\author{作者名}
\date{\today}

\begin{document}

\maketitle

\section{章节标题}
这里是正文内容。

\subsection{小节标题}
更细一级的内容。

\end{document}
```

### 数学公式

```latex
行内公式：$E = mc^2$

独立公式（带编号）：
\begin{equation}
    \int_0^\infty e^{-x^2} \, dx = \frac{\sqrt{\pi}}{2}
\end{equation}
```

### 列表

```latex
\begin{itemize}
    \item 无序列表项一
    \item 无序列表项二
\end{itemize}

\begin{enumerate}
    \item 有序列表项一
    \item 有序列表项二
\end{enumerate}
```

### 表格

```latex
\begin{table}[h]
    \centering
    \begin{tabular}{|c|c|c|}
        \hline
        列1 & 列2 & 列3 \\
        \hline
        a & b & c \\
        \hline
    \end{tabular}
    \caption{表格标题}
    \label{tab:example}
\end{table}
```

### 插入图片

```latex
\begin{figure}[h]
    \centering
    \includegraphics[width=0.8\textwidth]{image.png}
    \caption{图片说明}
    \label{fig:example}
\end{figure}
```

### 交叉引用与参考文献

```latex
如图~\ref{fig:example}所示……

参考文献引用：\cite{author2024}
```

`\label` 打标签、`\ref` 引用编号，这一套组合可以用在图、表、公式、章节上；`~` 是不换行空格，避免"如图"和编号被拆到两行。参考文献配合 `.bib` 文件和 BibTeX / BibLaTeX 使用，编译几次之后引用列表和编号会自动生成、自动排序。

## 小结

想快速体验就先上 Overleaf，不用装任何东西；想要更顺手的本地工作流，再根据自己的使用习惯选传统编辑器全家桶，或者 VS Code + LaTeX Workshop。语法一开始会有点不适应，但写惯了之后，尤其是涉及大量公式和引用的文档，会觉得比 Word 顺手太多。
