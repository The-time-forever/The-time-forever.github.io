C 盘老是爆红、桌面图标堆得找不到东西、D 盘里一堆叫"新建文件夹"的文件夹——这几乎是每个 Windows 用户都会遇到的问题。其实解决办法不复杂，核心就是养成一套简单的分区习惯，并且理解清楚"快捷方式"和"文件本体"的区别。这篇文章把我自己长期在用的整理方式写下来，供参考。

## 1. C 盘为什么老是爆红，怎么办

C 盘会越用越满，主要是这几个原因叠加：Windows 系统更新缓存、各种软件默认把安装路径、缓存、临时文件写在 C 盘、以及用户自己没注意把东西存到了 C 盘。

**如果你要写代码，C 盘最好留够 400GB 以上的空间。** 这不是危言耸听——很多开发工具（比如 Docker 镜像、各种语言的全局依赖缓存、IDE 索引文件）即使你把"安装路径"选在了 D 盘，实际运行时产生的大量缓存、虚拟环境、编译产物依然会写回 C 盘的用户目录或系统目录下。反过来，有些编程工具如果被强行拖去装在 D 盘，会因为程序内部写死了对系统盘路径的假设，出现各种难以预测、难以排查的 bug。所以更稳妥的做法是：**C 盘一开始就多留一些空间，而不是等到快满了才手忙脚乱地搬家。**

如果已经分好区、发现 C 盘太小想从 D 盘"匀"一点空间过来，也不是想象中那么简单。Windows 自带的磁盘管理（`diskmgmt.msc`）调整分区大小时有个方向限制：**只能把空间从前面的分区让给后面的分区**，也就是缩小 C 盘、把腾出来的空间扩展给 D 盘是可行的；但反过来，想缩小 D 盘、把空间匀给前面的 C 盘，用自带工具是做不到的——因为缩小 D 盘释放出来的空间只会出现在 D 盘末尾，并不挨着 C 盘，系统自带工具没法把两者合并。如果确实需要把 D 盘的空间"倒流"回 C 盘，只能借助第三方分区工具（比如 DiskGenius、EaseUS Partition Master、MiniTool Partition Wizard），它们能真正移动分区位置来腾出连续空间。这也是为什么前面说"C 盘一开始就要多留空间"——分区这件事，一步到位远比事后补救省心。

如果真的要动手操作，我自己用过 **[DiskGenius](https://www.diskgenius.cn/)** 和 **[傲梅分区助手](https://www.disktool.cn/)（AOMEI Partition Assistant）** 这两款工具。DiskGenius 界面是全英文的，对不熟悉英文界面的用户可能不太友好，但论功能完整度，我感觉它比傲梅还要更胜一筹。

需要提醒的是，这类跨盘调整分区的操作本身是有风险的，过程中容易遇到各种意外状况，所以动手之前最好先插好电源、别中途断电，重要数据也一定要提前备份。这件事多少因人而异——我在网上看到过不少人分区时踩坑，自己用傲梅分区助手也遇到过一次报错，好在最后分区还是成功了；后来就改用了 DiskGenius。我自己验证过一个相对稳妥的做法：先用第三方工具在 C、D 盘之间腾出一块**未分配**的空余空间（不要让它直接自动合并进 C 盘），再用 Windows 自带的磁盘管理把这块空余空间合并进 C 盘。这个流程看起来有点绕，但比让第三方工具一步到位地直接合并，更能避免分区之后开机进 PE 检测报错之类的奇怪问题。

当然，也可以干脆不分区，把整块硬盘都当 C 盘用，这样自然不存在"哪个盘不够用"的问题。但系统盘和个人文件混在一起，一旦系统出问题需要重装，所有文件也会一起受影响，所以除非硬盘容量本身很小、分不分意义不大，一般还是不建议这样做——这也是为什么大多数人依然会选择分盘。

这里要特别提醒一点：前面说的这些空间腾挪，都只能发生在**同一块物理硬盘**内部的分区之间——C 盘和 D 盘如果本来就在同一块硬盘上，才谈得上互相"匀"空间；如果它们分别是两块独立的物理硬盘，空间是没法直接划过去的，该扩容还是得给对应那块盘单独加装或更换硬盘。

日常维护上，C 盘的空间占用也不能全指望"一次留够"，还需要定期清理：Windows 自带的「磁盘清理」（`cleanmgr.exe`）或较新系统里的「存储感知」（设置 → 系统 → 存储）可以清理系统更新残留、临时文件和回收站；此外 `%temp%` 环境变量指向的目录（一般是 `C:\Users\你的用户名\AppData\Local\Temp`）也值得手动看一眼——很多软件运行时产生的临时文件会一直堆在这里，不会自动清除。

根据我自己的经验，`%temp%` 里的文件放心删就好，基本不会出问题；如果个别文件提示无法删除，直接跳过即可——绝大多数情况只是因为它正被某个进程占用，不是什么大问题。

推荐一个可视化查看空间占用的小工具：**[TreeSize](https://www.jam-software.com/treesize)**（JAM Software 出品），能清楚地看到到底是哪些文件夹在吃空间，比 Windows 自带的存储感知直观得多，免费版足够日常使用。

## 2. 桌面：宁愿整洁，也别什么都往上堆

### 理解快捷方式，而不是把 exe 拖上桌面

有些人会把程序的 `.exe` 文件直接拖到桌面，这其实是把**程序本体**移动到了桌面，而不是创建了一个指向它的引用。这样做至少有两个问题：一是原来的安装目录可能因此少了关键文件导致程序运行异常；二是桌面（`C:\Users\你的用户名\Desktop`）本身就是 C 盘的一部分，桌面上堆的文件越多，C 盘占用也在悄悄增加。

正确的做法是**创建快捷方式**：右键程序的 `.exe` → "发送到" → "桌面快捷方式"，或者在拖动的同时按住 `Alt` 键，松手时系统会自动创建一个快捷方式而不是移动文件本体。快捷方式本身体积很小，删掉它也不会影响原程序。

### 尽量不要把文件夹放在桌面

文件夹一旦扔上桌面，很容易演变成"什么都往里塞"的杂物堆，而且文件夹里的内容也是实打实占用 C 盘空间的。更好的做法是把文件夹放在 D 盘对应的位置（下一节会讲怎么组织），桌面只留必要的快捷方式或者近期正在处理的一两个临时文件。

如果你确实想保持桌面绝对清爽，也可以完全不用快捷方式，靠自己记住安装路径手动去启动 exe——但说实话这对大多数人来说太麻烦了。**在桌面上保留少量必要的快捷方式，是效率和整洁之间一个合理的折中**，不用因为追求"极简桌面"而束缚自己。


### 想更彻底一点？把桌面路径整体迁移到 D 盘

如果还是不放心桌面偶尔被人（或者自己手滑）扔进大文件占用 C 盘空间，可以干脆把"桌面"这个特殊文件夹的实际存储位置迁移到 D 盘——操作系统层面的桌面体验完全不变，图标位置、快捷方式照旧，只是背后存储的物理位置换了个盘：

1. 打开文件资源管理器，进入 `此电脑 → C:\Users\你的用户名`
2. 找到「桌面」文件夹，右键 → 属性
3. 切换到「位置」选项卡，把路径改成你想要的 D 盘目录，比如 `D:\Desktop`
4. 点击「移动」，系统会提示是否把现有桌面文件一并搬过去，选择"是"

这样以后哪怕不小心往桌面扔了大文件，占用的也是 D 盘空间而不是 C 盘。同样的方法也适用于 `C:\Users\你的用户名` 下的其他几个系统特殊文件夹：「文档」对应 `Documents`、「下载」对应 `Downloads`、「图片」对应 `Pictures`、「视频」对应 `Videos`、「音乐」对应 `Music`——中文只是资源管理器里显示的名字，实际文件夹名是英文。如果你习惯把下载的东西一直堆在默认的"下载"（`Downloads`）文件夹里不整理，也可以用同样的方式把它迁移到 D 盘，从源头上减少 C 盘被占用的可能。

> **补充一点小知识：** 这些特殊文件夹为什么在资源管理器里显示中文名，硬盘上的实际文件夹名却是英文？原因是每个特殊文件夹里都藏着一个隐藏的 `desktop.ini` 文件，里面写着类似 `LocalizedResourceName=@%SystemRoot%\system32\shell32.dll,-21770` 这样一行配置，指向系统 DLL 里按当前系统语言存好的一条文本资源。资源管理器读取到这个配置后，就会用查到的本地化名称（比如"下载"）替换掉显示效果，但文件夹在磁盘上的真实名字（`Downloads`）从始至终没有变过——这也是为什么你在地址栏直接敲英文路径 `C:\Users\用户名\Downloads` 也能进去，跟点"下载"这个中文名进去是同一个地方。

> 迁移过程中不要中途断电或强行关闭资源管理器，避免文件移动到一半出现异常；如果桌面文件本身比较多，移动会需要一点时间，耐心等它跑完就好。

还有一点要提醒：在「位置」选项卡里手动输入新路径时，字母（包括大小写）一定要输对，不要打错、漏字母。万一真的填错了也不用慌——回到同一个「位置」选项卡，点「还原默认值」就能把桌面路径恢复回最初的样子，重新填一遍即可。

## 3. D 盘的组织方式：两个顶层文件夹就够了

我个人习惯把 D 盘分成两大块：

- **`SoftWare/`** —— 存放所有安装的软件
- **`Document/`** —— 存放自己的文件

关键是要**善用子文件夹分类整理，不要什么都堆在根目录**。安装软件、修改安装路径到 D 盘时：

- 如果安装包本身比较智能，会自动在 `SoftWare/` 后面帮你建好子文件夹（比如自动生成 `SoftWare/VS Code/`），这种情况不需要自己再手动创建
- 如果安装包比较"傻"，直接把文件散落在你指定的目录里，那就需要自己提前建好一个子文件夹作为安装路径，命名直接用这个软件的英文名即可，最直观（比如 `SoftWare/PotPlayer/`）

`Document/` 下同理，也建议按用途再分一层子文件夹，比如：

```
D:\Document\
  Projects\      # 正在写的代码 / 课程作业
  Notes\         # 笔记、文档
  Media\         # 图片、视频素材
  Archive\       # 用不上但舍不得删的旧文件
  Downloads\     # 长期需要保留的下载内容
```

原则很简单：**看到一个文件不知道该放哪，就是该新建一个子文件夹的信号**，而不是随手扔进最外层了事。杂物堆得越久，以后想整理清楚的成本就越高。

## 4. 英文路径为什么这么重要

鉴于不少老牌程序、命令行工具、编译器对中文路径的兼容性并不好，很容易出现乱码、莫名其妙的报错，甚至程序直接崩溃。所以有几个地方建议一开始就用英文：

- **电脑的用户名**（也就是 `C:\Users\` 下面那个文件夹名）
- **电脑的设备名**
- **C 盘上涉及到的关键路径**，比如各种开发工具的安装目录
- **PATH 等环境变量**里引用到的路径

想知道自己电脑现在用的是不是中文用户名、设备名，打开 PowerShell（`Win + X` → 「Windows 终端」），用下面几条命令就能快速查清楚：

```powershell
$env:USERNAME       # 当前登录的用户名
$env:USERPROFILE    # 用户文件夹的完整路径，能直接看出用户名是不是中文
$env:COMPUTERNAME   # 电脑的设备名（计算机名）
```

设备名如果想改，最简单的方式是走图形界面：设置 → 系统 → 关于 → 「重命名这台电脑」，输入新的英文名后重启生效；也可以用管理员权限打开 PowerShell 执行：

```powershell
Rename-Computer -NewName "NewDeviceName" -Restart
```

这条命令同样需要重启才会生效，执行前记得先保存好手头的工作。

用户名就没这么简单了。如果电脑已经在用中文用户名，重新命名系统用户名操作起来比较繁琐、容易出问题，一般不建议直接改。更稳妥的办法是后续新建一个使用英文用户名的本地账户来专门做开发相关的事情，把开发工具、代码仓库都放在这个账户下操作，避免中文路径带来的坑。

这里要特别说明一下为什么不建议直接改用户名或者用户文件夹名：系统内部大量注册表项、已经写死的 PATH 以及各种软件的配置文件，引用的都是你最初创建账户时的那个名字，并不会因为你改了显示名称就跟着同步更新。贸然把用户名或文件夹名改掉，只会让原本能正常工作的路径大面积失效，报错只多不少，而且要把这些散落各处的引用一条条揪出来改干净几乎不现实——所以这里不建议、也不展开讲具体怎么改，遇到这种情况，前面说的"新建一个英文用户名的本地账户"才是更省心的路。

值得一提的是，Windows 11 在这方面已经改善了不少：如果电脑一开始就是用微软账户联网激活的（现在不少轻薄本出厂设置就是如此），系统会直接取邮箱地址前几位作为用户名，天然是英文，完全不用操心这个问题。但如果是自己组装的电脑，或者激活时特意跳过了微软账户登录、选择了本地账户（这里不太推荐这种方式——一来错过云端同步的便利，二来就容易踩中用户名这个坑），设置本地账户时就一定要记得输入英文用户名，也尽量别用空格或者奇怪的特殊字符。

另外一个经常被忽略的细节：**路径里尽量不要包含空格**。不少命令行工具在处理带空格的路径时需要额外加引号才能正确识别，一不小心就会报错。文件夹命名时统一用连字符 `-` 或下划线 `_` 代替空格，可以省掉很多不必要的麻烦。

### 对比一下：macOS 在这方面天生就少很多烦恼

如果你同时也在用 Mac，会发现前面说的"C 盘不够用""路径必须用英文"这些烦恼，在 macOS 上基本不存在。原因也不复杂：macOS 底层是 Unix 体系，文件系统对多字节字符（包括中文）的支持一直比较完善，命令行工具和开发环境大多也是在 Unix/Linux 生态里成长起来的，天然不排斥中文路径；再加上 macOS 没有 Windows 这种把系统、程序、用户数据都塞进同一个"C 盘"的习惯，存储空间是作为一整块统一管理的，也就不存在"系统盘被各种缓存挤爆、想从别的分区匀点空间过来还得靠第三方工具"这种情况。

所以在 Mac 上，基本不用操心分区、路径编码这些底层问题，唯一需要花心思的就是把自己的数据文件（文档、项目、下载）分类组织好。这其实也印证了整篇文章的核心思路：**不管在哪个系统上，整理电脑说到底都是在管理好自己的文件**，只是 Windows 用户还得额外应付几个系统本身带来的麻烦。

## 5. 比起大扫除，更重要的是养成习惯

一次性整理干净不难，难的是维持住。比起等到整个目录乱得没法看了才手忙脚乱地清理，更好的方式是平时就顺手做好这几件小事：

- 每隔一段时间清空一下**下载文件夹**，该转移的转移、该删的删
- 新建文件或文件夹时不要嫌麻烦，随手分好子文件夹归类，而不是先扔在最外层"回头再说"
- 用 **TreeSize** 定期看一眼空间占用大户，及时发现异常增长的缓存目录
- 打开 Windows 的「存储感知」（设置 → 系统 → 存储），让系统自动清理临时文件和回收站

整理电脑本质上和整理房间是一回事：分类明确、随手归位，比事后集中大扫除轻松得多。
